## 🛠️ 所有人第一步：本地环境搭建 + 数据入库

> ⚠️ 这一步每个人都必须做，否则看不到真实数据。

### 1. 拉取最新代码

```bash
cd ~/FoundationPitSafetyManagement-System
git pull origin main
```

### 2. 安装 Python 依赖（生成导入SQL需要）

```bash
pip3 install pandas openpyxl
```

### 3. 确保 MySQL 8.0 已启动，创建数据库

```bash
# 登录 MySQL（密码改成你自己的）
mysql -u root -p
```

```sql
CREATE DATABASE IF NOT EXISTS pit_safety_db DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
exit;
```

### 4. 建表 + 导入数据（按顺序执行）

```bash
cd ~/FoundationPitSafetyManagement-System

# 第一步：生成导入 SQL 文件（读取 sql/data/ 下的原始CSV/Excel）
python3 sql/import_data.py

# 第二步：建表 + 种子数据
mysql -u root -p pit_safety_db < sql/init_schema.sql

# 第三步：按文件大小从小到大导入
mysql -u root -p pit_safety_db < sql/import_adjust.sql          # 调整记录(小)
mysql -u root -p pit_safety_db < sql/import_total_station.sql   # 全站仪(4MB)
mysql -u root -p pit_safety_db < sql/import_steel_temp.sql      # 钢支撑温度(6MB)
mysql -u root -p pit_safety_db < sql/import_axial_force.sql     # 伺服轴力(9MB)
```

### 5. 验证数据是否导入成功

```sql
mysql -u root -p pit_safety_db -e "
SELECT 'steel_temp' AS tbl, COUNT(*) FROM data_steel_temperature
UNION ALL SELECT 'total_station', COUNT(*) FROM data_total_station
UNION ALL SELECT 'axial_force', COUNT(*) FROM data_axial_force
UNION ALL SELECT 'adjust', COUNT(*) FROM maintenance_adjust_record;
"
```

**预期输出**：

```
steel_temp      122127
total_station   46622
axial_force     181527
adjust          1449
```

### 6. 修改后端数据库密码

打开 `backend/src/main/resources/application.yml`，把 `password` 改成你自己的 MySQL 密码：

```yaml
spring:
  datasource:
    password: 你的密码
```

## 👤 吴博（后端）— 工单系统 API + 设备管理 API 补全

### 本周任务

#### 任务 A：工单全链路 API

新建包 `cn.edu.cdu.pitsafety.workorder`

| 接口                                  | 说明                   | 状态变化                |
| ------------------------------------- | ---------------------- | ----------------------- |
| `GET /api/work-order/list`          | 工单列表（筛选+分页）  | —                      |
| `GET /api/work-order/{id}`          | 工单详情（含关联信息） | —                      |
| `POST /api/work-order/create`       | 创建工单               | **新增** status=0 |
| `PUT /api/work-order/{id}/assign`   | 指派维修工             | 0→1                    |
| `PUT /api/work-order/{id}/start`    | 开始维修（到场签到）   | 1→2                    |
| `PUT /api/work-order/{id}/complete` | 完成维修填日志         | 2→3                    |
| `PUT /api/work-order/{id}/accept`   | 验收通过               | 3→4                    |
| `PUT /api/work-order/{id}/cancel`   | 取消工单               | 仅0或1可取消            |

**状态流转图**：

```
待派单(0) → 待接单(1) → 维修中(2) → 待验收(3) → 已完成(4)
```

**开发提示**：

- 工单流水号规则：`WO` + 日期 + 3位序号，如 `WO20260512001`，实现时查当天最大序号 +1
- 状态流转必须校验前驱状态，不能跳跃（如不能从待派单直接跳到维修中）
- `repairer_id` 只有 REPAIRER 角色的用户才能被指派
- 用 `@Select` + XML 混合方式（简单查询用注解，关联查询用XML）

**新建文件清单**：

```
backend/src/main/java/cn/edu/cdu/pitsafety/workorder/

├── service/WorkOrderService.java
├── service/impl/WorkOrderServiceImpl.java
├── mapper/WorkOrderMapper.java
├── entity/WorkOrder.java
└── dto/WorkOrderCreateRequest.java

backend/src/main/resources/mapper/
```

**a) `request.js` 加 Token 拦截器**

在 `frontend/src/api/request.js` 中添加 request 拦截器：

```javascript
// 在 axios.create 之后加
request.interceptors.request.use(config => {
  const token = localStorage.getItem('token')
  if (token) {
    config.headers.Authorization = `Bearer ${token}`
  }
  return config
})
```
